# 📦 Fullstack Cargo Management System — Backend (Китай — Бишкек)

Backend-часть профессионального Fullstack-приложения для автоматизации логистических процессов карго-компании. API полностью сопровождает груз: от приемки на складе в Китае до выдачи клиенту в Бишкеке.

---

## 📝 Описание бизнес-логики (Бизнес-процесс)

Backend реализует API для 3 ключевых этапов жизненного цикла груза:

1. **🇨🇳 Этап 1: Приемка в Китае (Импорт данных)**
   - **Что делает:** Эндпоинты отвечают за прием и обработку Excel-файлов с информацией о товарах, поступивших на склад в Китае.
   - **Как работает:** Менеджер загружает `Excel-файл (.xlsx / .xls)` через API. Сервер парсит файл, валидирует данные и сохраняет их в базу данных, присваивая товарам начальный статус (например, `На складе в Китае`).

2. **🇰🇬 Этап 2: Склад в Бишкеке (Прибытие груза)**
   - **Что делает:** Эндпоинты фиксируют прибытие товаров в точку назначения.
   - **Как работает:** API принимает запросы на поиск товаров по трек-номерам, кодам или ID и обновляет статус в базе данных на `Прибыло в Бишкек / Готово к выдаче`.

3. **🤝 Этап 3: Выдача клиентам (Финал)**
   - **Что делает:** Эндпоинты отвечают за непосредственную передачу посылок клиентам.
   - **Как работает:** API находит товары клиента, проверяет готовность, списывает их со склада и отмечает как `Выдано`.

## ⚙️ Требования (Prerequisites)

Перед началом работы убедитесь, что у вас установлены:

- **Node.js** (рекомендуемая версия LTS, например, v20+)
- **npm** (идет в комплекте с Node.js)
- **PostgreSQL** (https://www.postgresql.org/download)

---

## Установка

1. Клонируйте репозиторий:
   ```bash
   git clone https://github.com/AzatBaisynov/ades-cargo-backend.git
   ```

1.1 Перейдите в вашу репозиторий

```bash
cd ваш репозиторий
```

2. Установите зависимости

   ```bash
   npm i
   ```

3. Создайте файл `.env` в корне папки backend и укажите необходимые переменные окружения:

   ```env
   PORT=3000

   DB_HOST=localhost
   DB_PORT=5432
   DB_USERNAME=postgres
   DB_PASSWORD=ваш_пароль

   ```

4. Установите необходимые библиотеки (если не установились через `npm i`):
   например:

   ```bash
   npm i @nestjs/typeorm typeorm pg @nestjs/config multer @nestjs/schedule class-validator class-transformer xlsx
   ```

5. Запуск приложения:

   ```bash
   # Режим разработки
   npm run start:dev

   # Production режим
   npm run start:prod
   ```

   🌐 Backend запустится на: (обычно http://localhost:3000).
